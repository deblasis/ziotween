# Contributing

## Development
1. Fork the repository
2. Create a feature branch
3. Make changes with tests
4. Run `zig build test` and `zig fmt`
5. Submit a pull request

## Code Style
- Follow Zig standard formatting (`zig fmt`)
- Document all public symbols with `///` comments
- Add tests for new functionality
- Keep the library zero-dependency

## AI Usage
AI-generated contributions are welcome. Please ensure code is correct, tested, and documented.
